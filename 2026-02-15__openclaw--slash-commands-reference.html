OpenClaw 슬래시 커맨드 레퍼런스

작성일: 2026-02-15 · 대상: 전문 개발자(가독성 우선) · 범위: OpenClaw Gateway slash commands
/status /model /think /reasoning /subagents /context

요약

  • OpenClaw의 슬래시 커맨드는 Gateway가 처리한다. (대부분 단독 메시지로 /...를 보내면 즉시 실행)
  • Directives(예: /think, /model)는 모델 호출 전에 해석되어 세션 설정을 바꾼다.
  • 일부 기능은 기본 비활성/권한 제한(예: /restart, /config, /debug, /bash)이 있을 수 있다.
주의: 그룹 채팅에서 /reasoning, /verbose는 내부 출력/도구 로그가 노출될 수 있어 위험하다. 필요할 때만 켜고, 기본은 끄는 편이 안전.

0) 공통 규칙 / 실행 방식

커맨드 vs 디렉티브

  • Commands: /status, /help 같은 “기능 호출”.
  • Directives: /think, /model, /exec 같은 “세션/모델 실행 옵션”.

지속성(persist) 규칙

  • 디렉티브만 있는 메시지로 보내면 세션에 지속된다. (예: /think high만 보내기)
  • 일반 문장 속에 섞어 쓰면 “힌트”로만 처리되고 지속되지 않을 수 있다. (예: 이거 요약해줘 /think high)

구문

  • /cmd arg1 arg2
  • /cmd: arg (콜론 허용)

빠른 확인

실제로 “지금 텔레그램에서 등록된 커맨드 목록”은 /commands로 확인하는 게 가장 확실하다.

1) 도움말/상태/정체성

커맨드역할(무슨 문제를 해결?)예시
/help 도움말(핵심 커맨드/사용법) 출력. 
/help
/commands 현재 환경에서 사용 가능한 커맨드 목록(기본 + 스킬)을 출력. “정답 목록” 확인용. 
/commands
/status 현재 세션 상태(모델/설정/사용량 등)를 요약해서 보여줌. 디버깅과 비용/토큰 확인에 유용.
  • 응답이 느려졌을 때: 큐/모델/연결 상태 점검
  • 모델이 바뀌었나? thinking/verbose가 켜졌나? 확인
 
/status
/whoami
(alias: /id)		 현재 사용자의 sender id(식별자) 출력. allowlist 설정/권한 문제 디버깅에 필수. 
/whoami
/id

2) 세션/대화 흐름 제어

커맨드역할예시
/stop 현재 진행 중인 작업(모델 응답/툴 실행 등)을 중단 요청. 
/stop
/reset
또는 /new [model]		 새 대화처럼 컨텍스트를 리셋(세션 초기화). /new는 모델 힌트를 같이 줄 수 있음.
모델 힌트는 “지정”이 아니라 “선호”로 동작할 수 있음(설정/가용 모델에 따라).
 
/reset
/new
/new openai/gpt-5.2
/send on|off|inherit 메시지 전달(발송) 정책 제어. (owner-only일 수 있음) 
/send off
/activation mention|always 그룹 채팅에서 “멘션 필요 여부” 같은 반응 조건 제어(그룹용). 
/activation mention

3) 모델/추론/출력 제어 (Directives)

Directives는 모델 호출 전에 해석되어, “어떻게 답할지”를 바꾼다.

디렉티브역할예시
/think off|minimal|low|medium|high|xhigh
aliases: /thinking, /t		 모델의 사고(추론) 강도를 조절. 복잡한 문제/코드/설계는 올리고, 단순 질의는 낮추는 게 비용-효율적.
“세션에 지속”시키려면 단독 메시지로 보내는 게 안전.
 
/think low
/think high
/reasoning on|off|stream
alias: /reason		 reasoning(추론 과정)을 별도 메시지로 노출(또는 텔레그램 드래프트 스트리밍). 디버깅용.
그룹에서 위험: 내부 정보/도구 출력이 섞일 수 있어 기본 off 권장.
 
/reasoning on
/reasoning off
/reasoning stream
/verbose on|full|off
alias: /v		 도구 호출/진행 상황 등 “자세한 출력”을 늘림. 디버깅할 때만 켜는 편이 안전. 
/verbose on
/verbose off
/model <name>
alias: /models		 사용할 모델을 선택/변경. /model만 치면 picker/list가 뜰 수 있음.
  • 품질/가격/속도 튜닝
  • 특정 작업에 강한 모델로 스위치
 
/model
/model list
/model openai/gpt-5.2
/queue <mode> 메시지 처리 큐 정책 설정(디바운스/캡/드랍 전략 등). 입력이 몰릴 때 UX 조절.
상세 옵션은 환경/버전에 따라 다르므로 /queue 단독 실행으로 현재 설정 확인 권장.
 
/queue
/queue debounce:2s cap:25 drop:summarize
/usage off|tokens|full|cost 응답 하단에 토큰/비용/사용량 표시를 켜거나 끔. 비용 민감하면 on. 
/usage tokens
/usage off
/tts ... TTS(음성 출력) 정책 설정. (off/always/inbound/tagged 등) 
/tts status
/tts off

4) 도구 실행/권한 (고급)

커맨드역할예시
/elevated on|off|ask|full
alias: /elev		 위험한 exec/외부작업 승인 정책을 조절. full은 승인 없이 실행할 수 있어 위험.
보안상 기본은 ask 또는 off 권장.
 
/elevated ask
/elevated off
/exec host=... security=... ask=... node=... exec 실행 대상/보안 모드 설정. 보통은 현재 설정 조회용으로도 씀.
예: sandbox/host/node 어디에서 쉘을 실행할지, allowlist/full 같은 보안 레벨.
 
/exec
/exec host=gateway security=allowlist ask=on-miss
/approve <id> allow-once|allow-always|deny exec 승인 프롬프트(approval) 처리. “어떤 명령을 허용할지” 명시적으로 결정. 
/approve 123 allow-once
/approve 123 deny
/allowlist 허용 목록 조회/추가/삭제. 커맨드/exec 권한 문제 해결에 사용. 
/allowlist

bash 실행 (옵션/권한 필요)

아래는 환경 설정에 따라 비활성일 수 있다. (commands.bash: true + elevated allowlist 등)
커맨드역할예시
/bash <command>
alias: ! <command>		 호스트에서 쉘 커맨드를 실행. 빠른 진단/파일 조작/스크립트 실행. 
/bash ls -la
! ls -la
!poll 백그라운드로 돌고 있는 bash 작업의 출력/상태를 확인. 
!poll
!stop 실행 중인 bash 작업을 중지. 
!stop

5) 컨텍스트/서브에이전트/스킬 실행

/context

  • /context list: 현재 컨텍스트 구성(파일/도구/스킬) 요약
  • /context detail: 각 항목별 크기/구성 등 상세(토큰 소비 분석용)
  • /context json: 기계가 읽기 쉬운 형태(자동화/로그용)
/context
/context detail

/subagents

서브에이전트(백그라운드 작업)를 점검/제어.

  • /subagents list: 실행 중/최근 서브에이전트 목록
  • /subagents log ...: 로그 확인
  • /subagents send ...: 서브에이전트에 메시지
  • /subagents stop ...: 중지
/subagents list

/skill <name> [input]

스킬을 이름으로 실행. (개별 스킬이 슬래시로 노출되지 않는 환경에서 특히 유용)

/skill weather 서울
/skill pdf-workflow <...>
스킬이 “슬래시 명령어로 바로 노출”되는지는 설정(nativeSkills) 및 스킬 메타(user-invocable)에 좌우된다.

6) 설정/디버그/재시작 (owner-only 또는 비활성일 수 있음)

보통 운영자(Owner)만 쓰거나, config에서 기능을 켜야 동작한다.

커맨드역할예시
/config show|get|set|unset on-disk config(openclaw.json)를 조회/수정. 잘못 건드리면 장애가 날 수 있으므로 신중.
기본 비활성일 수 있음: commands.config: true
 
/config show
/config get messages.responsePrefix
/debug show|set|unset|reset 런타임 오버라이드(메모리) 설정. 디스크에는 안 쓰고 즉시 적용(실험/진단용).
기본 비활성일 수 있음: commands.debug: true
 
/debug show
/debug set messages.responsePrefix="[openclaw]"
/restart Gateway/런타임 재시작. 기본 비활성일 수 있으며 운영 환경에서는 영향이 큼.
기본 비활성일 수 있음: commands.restart: true
 
/restart
