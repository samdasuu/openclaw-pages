클로드 코드 200% 활용하기: 서브 에이전트, 스킬, 커맨드 완벽 정리

출처: 유튜브 '코딩알려주는누나' - 영상 보기

안녕하세요! AI로 코딩하다 보면 "아, 내가 일일이 다 지시하고 검사해야 하나? 너무 귀찮은데..."라고 느끼신 적 있으시죠? 오늘은 여러분을 단순 작업자에서 'CEO'로 승진시켜 줄 클로드 코드(Claude Code)의 핵심 기능 3가지를 정리해 드립니다.

왜 이 기능들을 써야 하나요?

AI에게 매번 "이거 해줘, 저거 해줘" 시키는 건 비효율적입니다. (마치 사장님이 대리점 청소부터 회계까지 다 하는 꼴이죠.)
대신 유능한 직원(서브 에이전트)을 고용하고, 업무 매뉴얼(스킬)을 주고, 간단한 결재(커맨드)만 하면 일이 훨씬 빨라집니다.

1. 서브 에이전트 (Sub-Agent): 나만의 전문가 직원들

🏢 비유: 각 분야별 전문가 직원

내가 모든 걸 다 하는 '1인 기업'에서 벗어나세요. 기획팀장, 개발팀장, QA팀장을 따로 두는 겁니다.

서브 에이전트는 특정 역할에 특화된 AI입니다. 예를 들어, 어떤 에이전트는 "창의적인 기획"만 잘하고, 어떤 에이전트는 "버그 없이 코드 짜기"만 잘하도록 설정할 수 있습니다.

  • 사용법: /agent create 명령어로 생성
  • 활용 예시:
    • Planner Agent: 요구사항을 듣고 꼼꼼한 기획서를 작성하는 직원
    • Coder Agent: 기획서를 보고 실제 코드를 구현하는 직원
    • Tester Agent: 코드를 보고 버그가 없는지 깐깐하게 검사하는 직원

이렇게 역할을 나누면, AI가 서로의 결과물을 검토(크로스 체크)하게 되어 코드 퀄리티가 훨씬 좋아집니다.

2. 스킬 (Skills): 우리 회사의 업무 매뉴얼

📘 비유: 사내 규정 / 업무 매뉴얼

"우리 회사는 보고서 쓸 때 꼭 결론부터 씁니다." 처럼, 직원들이 일을 할 때 지켜야 할 공통 규칙입니다.

에이전트들이 작업을 수행할 때 반드시 참고해야 하는 규칙이나 도구들을 정의한 것입니다. .clauderc 파일이나 skills/ 폴더에 저장해 두면 됩니다.

  • 활용 예시:
    • "모든 코드에는 반드시 한글 주석을 달아라."
    • "변수명은 camelCase로 작성해라."
    • "에러 발생 시 슬랙으로 알림을 보내라."

매번 "한글 주석 달아줘"라고 말할 필요 없이, 스킬로 등록해두면 알아서 척척 해냅니다.

3. 커맨드 (Commands): 업무 지시 단축키

⚡ 비유: 업무 지시 단축키 / 결재 서류 양식

사장님이 매번 구구절절 설명하지 않고 "김 대리, [프로젝트 A] 진행시켜!"라고 한 마디만 하면 끝나는 것과 같습니다.

자주 사용하는 긴 명령어들을 짧은 단축키 하나로 묶는 기능입니다. 복잡한 과정을 한 번에 실행할 수 있습니다.

  • 활용 예시: /develop 이라고 치면?
    1. → 기획 에이전트가 기획서 작성
    2. → 개발 에이전트가 코드 구현
    3. → 테스트 에이전트가 검수
    이 모든 과정이 자동으로 연달아 실행됩니다.

4. 추천 도구: BKit (비키트)

영상에서는 'BKit'이라는 유용한 패키지를 추천합니다. 이미 잘 만들어진 서브 에이전트와 워크플로우를 모아둔 '종합 선물 세트' 같은 것입니다.

💡 초보자용 해설: PDCA 사이클이란?

Plan (계획) → Do (실행) → Check (검토) → Act (개선)
일을 잘하기 위한 4단계 반복 과정입니다. BKit을 쓰면 이 과정을 AI가 알아서 돌려줍니다. (계획 짜고, 만들고, 검사하고, 고치고!)

개발자가 아니더라도 이런 도구들을 활용하면, 마치 유능한 개발팀을 거느린 CEO처럼 멋진 결과물을 만들어낼 수 있습니다. 지금 바로 시작해 보세요! 🚀
